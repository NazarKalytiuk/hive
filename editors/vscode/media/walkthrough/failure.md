# Inspecting a failure

When a step fails, the extension renders a rich `TestMessage` with:

- Expected vs actual with a unified diff.
- Structured **failure category** (`assertion_failed`, `connection_error`, `timeout`, `parse_error`, `capture_error`, `unresolved_template`).
- Stable **error code** (13 codes — see the schema).
- Full **request** (method, URL, headers, body) — redacted.
- Full **response** (status, headers, body) — redacted.
- **Remediation hints** generated by Tarn.

## Diagnosis loop

1. Branch first on `failure_category` — it tells you whether the issue is your assertion, the network, the test config, or the application.
2. Then read the specific `error_code`.
3. Then look at `assertions.failures` for the mismatched value.
4. Then optionally inspect the captured `request` / `response` to understand what the server actually did.

## Secrets

Any header matching `authorization`, `cookie`, `set-cookie`, `x-api-key`, `x-auth-token` is redacted to `***`. Add more header names via the `tarn.redactionExtraHeaders` setting.
